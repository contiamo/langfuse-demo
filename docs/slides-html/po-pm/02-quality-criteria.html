You can't improve
what you can't measure.
'Make it more helpful' isn't shippable. The PM's job is to turn it into a number you can track — and defend.
The problem
Why 'be more helpful' isn't shippable
It's a feeling, not a signal Two people will disagree on whether it improved
You can't catch a regression Did that prompt change quietly make it worse?
You can't compare variants Model A vs B, prompt v1 vs v2 — on what basis?
Acceptance criteria don't fit 'Click X → see Y' breaks when output is probabilistic
The PM's definition-of-done: quality criteria + a dataset + a threshold.
The building block
Anatomy of a score dimension
Name faithfulness, completeness, tone, safety
Type numeric 0–1 · boolean · categorical
What it rewards the behaviour you want more of
What it penalises the failure you're hunting
Who scores it a human, a code check, or an LLM judge — same table
Define it once; every annotator and every judge writes to the same dimension.
Worked example
A starter set — the Sherlock demo
Faithful
Grounded in the source
numeric 0–1
  • Only claims the context supports
  • No invented facts
Complete
Answers the whole question
numeric 0–1
  • Doesn't dodge parts
  • Covers what was asked
Tone
Fits the product voice
categorical
  • good / off / wrong
  • Matches the brand
Safety
Nothing harmful
boolean
  • In scope, appropriate
  • No unsafe content
Feedback
The user's verdict
0 or 1
  • 👍/👎 from the app
  • Real signal, zero cost
For Cornelsen

Language quality Answers hold up across German and English
